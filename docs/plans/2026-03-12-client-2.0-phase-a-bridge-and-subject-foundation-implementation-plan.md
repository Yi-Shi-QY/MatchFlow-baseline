# Client 2.0 Phase A Bridge And Subject Foundation Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Shrink the default manager bridge path and establish a subject-neutral execution shell so the client can validate a second runtime pack on a cleaner 2.0 core.

**Architecture:** Keep the current manager gateway and runtime-domain-pack architecture as the single source of truth. In this phase, do not introduce a second runtime pack yet. First move Command Center farther toward projection-first gateway contracts, then extract the minimal subject-neutral shell from the current match-first analysis page and runtime flow.

**Tech Stack:** React 19, TypeScript, Vite, Vitest, Capacitor, manager gateway projection model, subject route `/subject/:domainId/:subjectId`, local SQLite with localStorage fallback.
