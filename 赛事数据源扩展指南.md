# 赛事数据源扩展指南 (Match Data Extension Guide)

本文档以 **"添加比赛赔率 (Odds)"** 为例，详细说明如何在现有的 MatchFlow 架构中扩展赛事数据。

我们将涉及三个层面的修改：
1. **数据库层**: 修改 Schema 以存储新字段。
2. **服务端层**: 更新 API 以接收并返回新字段。
3. **客户端层**: 更新 TypeScript 类型定义并在 APP UI 中显示新数据。

---

## 1. 数据库层修改 (Database)

我们需要在 `matches` 表中存储赔率数据。由于赔率结构可能变化（不同博彩公司、不同盘口），推荐使用 PostgreSQL 的 **JSONB** 类型，它既灵活又支持索引。

### 操作步骤

连接到你的 PostgreSQL 数据库（可以使用 pgAdmin 或命令行），执行以下 SQL 语句：

```sql
-- 1. 添加 odds 字段，类型为 JSONB，默认为空对象
ALTER TABLE matches ADD COLUMN odds JSONB DEFAULT '{}'::jsonb;

-- 2. (可选) 如果你需要经常按赔率查询（例如查询主胜赔率大于 2.0 的比赛），可以建立 GIN 索引
CREATE INDEX idx_matches_odds ON matches USING GIN (odds);
```

**预期的数据结构示例**:
```json
{
  "had": { "h": 1.5, "d": 3.5, "a": 5.0 }, // 胜平负
  "hhad": { "h": 2.0, "d": 3.2, "a": 3.5, "goalline": -1 } // 让球胜平负
}
```

## 2. 服务端层修改 (Server)

我们需要修改 `match-data-server/index.js`，以便 API 能够读写这个新字段。

### 2.1 修改写入接口 (`POST /admin/matches`)

找到 `app.post('/admin/matches', ...)` 路由处理函数。

1.  **获取参数**: 在 `req.body` 解构中添加 `odds`。
2.  **修改 SQL**: 在 `INSERT` 和 `UPDATE` 语句中加入 `odds` 字段。

**代码修改示例**:

```javascript
// ... inside app.post('/admin/matches', ...)

const { 
  id, league_name, match_date, status, 
  home_team_id, away_team_id, 
  home_score, away_score, match_stats,
  odds // <--- [新增] 获取 odds 参数
} = req.body;

// ...

if (id) {
  query = `
    INSERT INTO matches (
      ..., match_stats, odds -- <--- [新增] 列名
    )
    VALUES (..., $9, $10)    -- <--- [新增] 占位符 (注意序号)
    ON CONFLICT (id) DO UPDATE SET
      ...,
      match_stats = EXCLUDED.match_stats,
      odds = EXCLUDED.odds   -- <--- [新增] 更新逻辑
    RETURNING *
  `;
  params = [
    ...,
    JSON.stringify(match_stats || {}),
    JSON.stringify(odds || {}) // <--- [新增] 传入参数
  ];
} else {
  // 同样修改 INSERT 逻辑 ...
}
```

### 2.2 修改读取接口 (`GET /matches`)

找到 `app.get('/matches', ...)` 和 `app.get('/matches/:id', ...)`。

PostgreSQL 的 `SELECT m.*` 语句会自动包含新添加的 `odds` 列。你只需要在 `formatted` 映射中将其透传给前端即可。

**代码修改示例**:

```javascript
const formatted = result.rows.map(row => ({
  id: row.id,
  league: row.league_name,
  // ...
  stats: row.match_stats,
  odds: row.odds, // <--- [新增] 将数据库字段映射到 API 响应
  // ...
}));
```

## 3. 客户端层修改 (Client App)

最后，我们需要在前端 APP 中定义类型并展示数据。

### 3.1 更新 TypeScript 类型定义

打开 `src/data/matches.ts`，找到 `Match` 接口定义，添加 `odds` 字段。

```typescript
export interface Match {
  id: string;
  league: string;
  date: string;
  status: 'live' | 'upcoming' | 'finished';
  // ...
  
  // [新增] 赔率字段 (可选)
  odds?: {
    had?: { h: number; d: number; a: number }; // 胜平负
    hhad?: { h: number; d: number; a: number; goalline: number }; // 让球胜平负
  };
}
```

### 3.2 更新 UI 展示

打开 `src/pages/MatchDetail.tsx` (或者 `Home.tsx`)，在合适的位置渲染赔率信息。

**代码修改示例**:

```tsx
{/* 在比分或时间下方显示欧赔 */}
{match.odds?.had && (
  <div className="mt-4 bg-zinc-900/50 p-3 rounded-lg border border-white/5">
    <div className="text-[10px] text-zinc-500 uppercase mb-2 text-center">胜平负 (HAD)</div>
    <div className="flex justify-between items-center text-xs font-mono">
      <div className="flex flex-col items-center w-1/3 border-r border-white/5">
        <span className="text-zinc-400 mb-1">主胜</span>
        <span className="text-emerald-400 font-bold">{match.odds.had.h}</span>

        <span className="text-zinc-400 mb-1">平</span>

      </div>
      <div className="flex flex-col items-center w-1/3">
        <span className="text-zinc-400 mb-1">客胜</span>

    </div>
  </div>
)}
```

## 4. 验证流程

1.  **数据库**: 执行 `ALTER TABLE` 语句。
2.  **服务端**: 重启 `match-data-server`。
3.  **数据注入**: 使用 Postman 或脚本向 `/admin/matches` 发送带有 `odds` 数据的 POST 请求。
    ```json
    {
      "league_name": "Premier League",
      "home_team_id": "...",
      "away_team_id": "...",
      "match_date": "...",
      "odds": {
        "eu": { "home": 1.8, "draw": 3.5, "away": 4.5 }
      }
    }
    ```
4.  **客户端**: 刷新 APP，进入比赛详情页，确认是否显示了赔率模块。

通过这种方式，你可以添加任意维度的数据（如：首发阵容、天气情况、裁判信息等），而无需重构整个系统架构。
