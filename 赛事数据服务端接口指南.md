# 赛事数据服务端接口指南

本文档定义了 MatchFlow 应用与外部赛事数据服务端的交互接口标准。开发者可以根据此文档实现自己的服务端，以便在应用中获取实时赛事数据。

## 1. 概述

MatchFlow 支持配置自定义的赛事数据源。用户在应用设置中输入 `Server URL` 和 `API Key` 后，应用将通过 HTTP 请求从该服务端获取数据。

- **协议**: HTTP/HTTPS
- **数据格式**: JSON
- **认证方式**: Bearer Token (通过 HTTP Header `Authorization: Bearer <API_KEY>`)

## 2. 接口定义

### 2.1 获取赛事列表

获取当前正在进行或即将开始的赛事列表。

- **Endpoint**: `/matches`
- **Method**: `GET`
- **Headers**:
  - `Authorization`: `Bearer <API_KEY>`
  - `Content-Type`: `application/json`

#### 请求参数 (Query Parameters)

| 参数名 | 类型 | 必填 | 描述 |
| :--- | :--- | :--- | :--- |
| `date` | string | 否 | 筛选日期，格式 `YYYY-MM-DD`。默认为当天。 |
| `status` | string | 否 | 筛选状态：`live` (直播中), `upcoming` (未开始), `finished` (已结束)。 |

#### 响应结构 (Response)

```json
{
  "data": [
    {
      "id": "string",           // 赛事唯一ID
      "league": "string",       // 联赛名称
      "date": "string",         // 比赛时间 (ISO 8601)
      "status": "string",       // 状态: "live" | "upcoming" | "finished"
      "homeTeam": {
        "id": "string",
        "name": "string",
        "logo": "string",       // 队徽 URL
        "form": ["W", "L", "D", "W", "W"] // 近期战绩 (可选)
      },
      "awayTeam": {
        "id": "string",
        "name": "string",
        "logo": "string",
        "form": ["L", "D", "L", "W", "L"] // 近期战绩 (可选)
      },
      "score": {                // 比分 (可选，直播或完场时需要)
        "home": number,
        "away": number
      },
      "stats": {                // 比赛数据 (可选)
        "possession": { "home": 50, "away": 50 },
        "shots": { "home": 10, "away": 8 },
        "shotsOnTarget": { "home": 4, "away": 3 }
      }
    }
  ]
}
```

### 2.2 获取赛事详情

获取单场赛事的详细信息。

- **Endpoint**: `/matches/:id`
- **Method**: `GET`
- **Headers**:
  - `Authorization`: `Bearer <API_KEY>`

#### 响应结构 (Response)

返回结构与列表中的单项结构一致。

```json
{
  "data": {
    "id": "string",
    "league": "string",
    // ... 其他字段同上
  }
}
```

## 3. 错误处理

服务端应返回标准的 HTTP 状态码：

- `200 OK`: 请求成功
- `401 Unauthorized`: API Key 无效或缺失
- `404 Not Found`: 资源不存在
- `500 Internal Server Error`: 服务端内部错误

错误响应体示例：

```json
{
  "error": {
    "code": "string",
    "message": "string"
  }
}
```

## 4. 开发示例 (Node.js/Express)

```javascript
const express = require('express');
const app = express();

const API_KEY = 'my-secret-key';

// 中间件：验证 API Key
const authenticate = (req, res, next) => {
  const authHeader = req.headers.authorization;
  if (authHeader === `Bearer ${API_KEY}`) {
    next();
  } else {
    res.status(401).json({ error: { message: 'Unauthorized' } });
  }
};

app.get('/matches', authenticate, (req, res) => {
  // 返回模拟数据
  res.json({
    data: [
      {
        id: '1',
        league: 'Premier League',
        date: new Date().toISOString(),
        status: 'upcoming',
        homeTeam: { id: 'h1', name: 'Arsenal', logo: '...', form: [] },
        awayTeam: { id: 'a1', name: 'Chelsea', logo: '...', form: [] }
      }
    ]
  });
});

app.listen(3000, () => console.log('Server running on port 3000'));
```
