# Match Data Server API Guide

This document is kept for compatibility with previous naming.

## Auth

All API routes except `/health` require:

`Authorization: Bearer <API_KEY>`

## Main Read Endpoints

- `GET /matches`
- `GET /matches/live`
- `GET /matches/:id`
- `GET /teams`
- `GET /teams/:id/matches`
- `GET /leagues`
- `GET /health`

## Admin Write Endpoints

- `POST /admin/init`
- `POST /admin/teams`
- `POST /admin/matches`
- `PUT /admin/matches/:id/score`
- `DELETE /admin/matches/:id`
- `DELETE /admin/teams/:id`

For deployment and environment setup, see `match-data-server/DEPLOY.md`.
